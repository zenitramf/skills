# PRD Template Reference

Use this as a quality check while writing the final PRD.

## Quality bar

- Write from the user's perspective first, then from the implementation perspective.
- Make user stories broad enough to cover normal flows, edge cases, permissions, operations, and recovery paths.
- Prefer implementation decisions that name responsibilities and contracts rather than code locations.
- Prefer tests that validate observable behavior.
- Keep out-of-scope explicit so the PRD stays actionable.
